# AQUA LUMINA 設定

## お問い合わせメール（スマホに届ける）

### 1. 受信アドレスを設定

`config.js` を開き、**スマホで受信しているメールアドレス**を入れます。

```js
contactEmail: 'あなたのスマホのメール@example.com',
```

※ 捨てメアド（tapi.re など）は FormSubmit から届きにくいです。

### 2. ローカルサーバーで開く

エクスプローラーから直接開く（`file://`）と送信できません。

```bash
npm start
```

ブラウザで **http://localhost:3456/contact.html** を開いて送信します。

### 3. 初回のみ：有効化（重要）

1. フォームから1回送信する
2. **スマホのメール**（迷惑メールフォルダも）で **「Activate Form」** を探す
3. メール内のリンクをタップして有効化する
4. もう一度フォームからテスト送信 → 問い合わせ内容が届く

### 4. 有効化後（任意・推奨）

有効化メールに載っている **`https://formsubmit.co/f/xxxxxxxx`** を
`contactFormEndpoint` に貼り付けます。

```js
contactFormEndpoint: 'https://formsubmit.co/f/xxxxxxxx',
contactEmail: '',
```

## 本番公開（デプロイ）

おすすめは **Render**（無料枠あり）。問い合わせフォーム・OGP もそのまま動きます。

### 手順

1. **GitHub にアップロード**
   - [GitHub](https://github.com/) で新しいリポジトリを作成
   - [GitHub Desktop](https://desktop.github.com/) で `test` フォルダを追加して Push  
     （`node_modules` は `.gitignore` で除外済み）

2. **Render でデプロイ**
   - [Render Dashboard](https://dashboard.render.com/) にログイン
   - **New +** → **Blueprint** → リポジトリを選択
   - `render.yaml` を読み込んで **Apply**

3. **環境変数を設定**（Render の Environment）
   - `SITE_URL` = デプロイ後の URL（例: `https://aqua-lumina.onrender.com`）  
     ※ OGP（SNSシェア画像）用

4. **公開 URL を確認**
   - 数分後 `https://xxxx.onrender.com` でサイトが開きます
   - 無料プランはしばらくアクセスがないとスリープし、初回表示が遅くなることがあります

### デプロイ後の問い合わせ

- `config.js` の `contactEmail` を本番用アドレスに設定
- デプロイ先 URL から `contact.html` でテスト送信
- 初回は FormSubmit の有効化メールが必要（ローカル時と同じ）

### 一時的に共有だけしたい場合

```bash
npm start
npm run share
```

`loca.lt` の URL が発行されます（ターミナルを閉じると無効）。
